Здесь описаны настройки тестов и теории.

Тесты

Тест может являться как самостоятельным электронным ресурсом, так и входить в состав электронного курса. При создании нового теста обязательно указывается его наименование.

1. На закладке «Вопросы» приведен перечень контрольных вопросов, входящих в состав теста, и указан вес каждого вопроса.

По клику на вопрос открывается форма вопроса, в которой указывается вес вопроса в данном тесте. Один вопрос может использоваться в разных тестах. В каждом из тестов вопрос может иметь различный вес, то есть поле «Вес вопроса» устанавливается в контексте теста.

Вес вопроса влияет на расчет итогового балла за тест. Например, в тесте из двух вопросов первый вопрос имеет вес равный 1, а второй – вес равный 2. Тогда итоговая оценка за тест при правильном ответе:

  • на все вопросы будет 100 баллов,
  • только на первый вопрос – 33 балла (1/3 от максимального балла),
  • только на второй вопрос –  67 баллов (2/3 от максимального балла), так как этот вопрос имеет больший вес.

Вопрос на закладку «Вопросы» можно добавить:

  • кнопкой «Создать», если вводится новый вопрос (см. раздел «Создание вопросов»),
  • кнопкой «Добавить из ранее созданных», если добавляется уже существующий вопрос,
  • кнопкой «Загрузить», если добавляются вопросы, созданные с использованием внешних редакторов (см. раздел «Загрузка электронных ресурсов из внешних файлов. Тесты»).

Вопросы можно объединять в папки. При тестировании каждая папка вопросов будет показана одной страницей теста, на которой последовательно будут расположены вопросы, входящие в папку. Порядок следования вопросов на странице теста соответствует порядку их указания в папке.

2. На закладке «Настройки» показаны многочисленные настройки, позволяющие создать тесты различного назначения.

Тест может быть предназначен для тренировки (упражнения) или для оценки знаний (аттестации). Для учета этой особенности предназначен параметр «Назначение», который может принимать значение «Аттестация» или «Упражнение». При выборе назначения «Упражнение» количество ответов пользователя на вопрос не ограничено. Если выбрано назначение «Аттестация», пользователь может ответить на каждый вопрос только один раз.

Параметр «Время теста» ограничивает время, отведенное для ответа на все вопросы теста. Если время не указано, то продолжительность прохождения теста не ограничивается.

Параметр «Количество попыток» определяет максимальное количество попыток прохождения данного теста, доступное пользователю. По умолчанию параметр равен нулю, что означает отсутствие ограничений на количество попыток выполнения теста.

Параметр «Количество страниц» задает максимальное количество страниц с вопросами, которые будут предложены пользователю в ходе тестирования. Если на закладке «Вопросы» папки отсутствуют, тогда количество страниц теста совпадает с количеством вопросов теста. В этом случае параметр задает количество вопросов теста, которые случайным образом выбираются программой из вопросов, указанных на закладке «Вопросы». Если параметр равен нулю (ноль - значение по умолчанию), то в тест выносятся все вопросы, указанные на закладке «Вопросы».

Флажок «Выбирать вопросы по темам рубрикаторов» устанавливает режим подбора вопросов в тест: из перечня вопросов, непосредственно указанных на закладке «Вопросы», или на основе рубрик и тем. При установке флажка закладка «Вопросы» меняет название на «Рубрики», а также изменяет свое содержание. Подробнее см. раздел «Выбор вопросов по темам рубрикатора».

Флажок «Исключить открытые вопросы» позволяет исключить из теста вопросы типа «Открытый», а также игнорировать настройку вопроса «Разрешен развернутый ответ». При установке флажка сформированные тесты точно не потребуют ручной проверки преподавателем, из каких бы вопросов не был составлен тест.

Флажок «Учитывать предыдущие попытки» позволяет управлять возможностью учета предыдущих ответов пользователя на вопросы. Если флажок установлен, то при подборе вопросов в тест предпочтение отдается вопросам, на которые пользователь еще не отвечал или отвечал неверно. Установку флажка удобно использовать, если требуется «прогнать» обучающихся по всем вопросам. Например, за семестр обучающиеся должны освоить обширную тему, по которой есть большое количество контрольных вопросов. В этом случае достаточно создать всего один тест, включающий все вопросы темы, и ограничить количество вопросов (параметр «Количество страниц») на каждую попытку. Если обучающийся будет проходить тест несколько раз (параметр «Количество попыток»), то в первый раз в тест попадут любые вопросы по теме. Во второй и последующие разы в тест попадут любые вопросы кроме тех, на которые в прошлые попытки он ответил верно.

Установка флажка «Показывать навигацию» позволяет отобразить пользователю перечень всех страниц данного теста. Если на одной странице теста размещен один вопрос (так бывает чаще всего), то пользователю показывается перечень всех вопросов назначенного ему теста. При снятом флажке пользователь видит только текущую страницу теста и не видит уже отвеченных им вопросов, а также страниц с вопросами, ожидающими ответа. Если флажок «Показывать навигацию» снят, то принудительно устанавливается флажок «Переходить вперед автоматически» (см. ниже).

С помощью флажка «Перемешивать» устанавливается случайность последовательности страниц с вопросами при каждой попытке тестирования. Если флажок снят, то вопросы будут предлагаться в порядке их указания на закладке «Вопросы». Если некоторые вопросы в ходе теста всегда должны идти друг за другом в определенном порядке, то эти вопросы следует поместить в одну папку на закладке «Вопросы». Тогда вне зависимости от флажка «Перемешивать» эти вопросы будут в неизменной последовательности отображаться на одной странице теста.

Флажок «Переходить вперед автоматически» устанавливает автоматический переход к очередному вопросу теста после ответа на текущий вопрос. Если флажок снят, то к следующему вопросу пользователь должен перейти самостоятельно с помощью навигации по тесту.

«Стартовая страница» позволяет задать текст с иллюстрациями и гиперссылками, который будет размещен на первой (стартовой) странице теста. На стартовой странице рекомендуется описывать назначение теста, давать методические указания по его прохождению.

3. На закладке «Комментарии к результату» указывается текст с иллюстрациями, который будет показан пользователю после выполнения теста. В зависимости от баллов, набранных за тест, и от использованных попыток тестирования могут быть показаны разные текстовые комментарии и картинки или звуковые файлы.

В верхней части формы указывается текст и файл, которые будут показаны всем пользователям вне зависимости от набранных баллов и использованных попыток тестирования. Например, слова «Благодарим за добросовестные ответы на вопросы, тест завершен!» и картинка букета цветов.

В нижней части формы можно создать несколько текстовых комментариев с картинками или звуковыми файлами, которые будут показаны пользователям, получившим за тест оценку в указанном диапазоне по 100-балльной шкале (параметры «От», «До» в поле «Диапазон баллов») и использовавшим при этом попытки тестирования, входящие в указанный диапазон (параметры «От», «До» в поле «Диапазон попыток»). По умолчанию параметры «От» и «До» заполнены нулями, что символизирует отсутствие ограничений.

При указании диапазонов следует обращать внимание, чтобы границы диапазонов различных комментариев не пересекались между собой. Например, если заданы два комментария и один комментарий показывается при баллах от 0 до 70, то второй должен показываться при баллах от 71 до 100.

4. На закладке «Сложности» указывается перечень различных уровней сложности теста, которые отличаются количеством страниц с вопросами и допустимым временем выполнения теста.

Если задан список уровней сложности, то следует указать наименование уровня сложности по умолчанию, настройки которого будут взяты из основных настроек теста. Перед выполнением теста обучающийся может самостоятельно выбрать уровень его сложности: оставить сложность по умолчанию или выбрать другой уровень из числа созданных на закладке «Сложности».

Тест может быть сформирован из вопросов, введенных на закладке «Вопросы», или путем выбора заданного количества вопросов из их тематических подборок, называемых далее темами. Управляет переходом к этому способу формирования теста флажок «Выбирать вопросы по темам рубрикаторов», доступный на закладке «Настройки». Например, в тест из пяти вопросов выбираются в случайном порядке три любых вопроса из одной темы и два вопроса из другой темы.

Темы вопросов объединены в рубрикаторы. Рубрикаторы и темы вопросов могут иметь различное методическое назначение, например, если в одной информационной базе работают преподаватели разных дисциплин, то рубрикаторы могут соответствовать названию дисциплин: «Высшая математика», «Общая физика», «Информационные технологии». В таком случае темами рубрикатора «Высшая математика» могут быть: «Определители», «Матрицы», «Системы линейных уравнений» и т.д. Если в информационной базе работают преподаватели одной дисциплины, то рубрикаторами могут быть, например, «Определители», «Матрицы» и «Системы линейных уравнений», а темами - «Простейшие вопросы», «Среднесложные вопросы», «Сложные вопросы».

Методический смысл использования тем рубрикаторов заключается в том, что единожды наполнив рубрикатор, методист сможет многократно и достаточно быстро формировать различные тесты, исходя из конкретных учебных задач. С другой стороны, при использовании тем рубрикаторов один тест при назначении его разным пользователям каждый раз будет иметь различный состав вопросов. Тогда тестируемые, например, сидящие в одном дисплейном классе и одновременно выполняющие один тест, не смогут списать друг у друга. Очевидным следствием из сказанного является необходимость иметь в теме рубрикатора существенно больше вопросов, чем выбирается из него в тест. Например, если в тест из темы рубрикатора выбирается 3 вопроса, то в этой теме рубрикатора желательно иметь более 5-7 вопросов. При использовании выбора вопросов по темам рубрикатора рекомендуется устанавливать флажок «Перемешивать», чтобы еще более индивидуализировать тест.

Для формирования теста путем выбора вопросов по темам рубрикатора нужно установить флажок «Выбирать вопросы по темам рубрикаторов», а на закладке «Рубрики» указать список рубрикаторов, их тем и количество вопросов, которое должно выбираться из каждой темы.

Параметр «Разрешенный вес» указывает вес вопросов, которые будут в случайном порядке выбираться в тест из темы рубрикатора. По умолчанию параметр равен нулю, что означает отсутствие ограничений по весу выбираемых вопросов. При установке параметра следует проверить, что в теме рубрикатора действительно есть достаточное количество вопросов с указанным весом. Следует обратить внимание, что если разные темы, добавленные в тест, содержат один и тот же вопрос, то в тест он будет добавлен только один раз. Тест не может содержать двух одинаковых вопросов.

Прежде чем создавать тесты подбором вопросов из тем рубрикаторов, требуется сформировать базу вопросов, создать рубрикаторы и их темы, а потом разместить в темы рубрикаторов необходимое количество вопросов. Один вопрос может использоваться в нескольких темах, причем в каждой из тем рубрикаторов он может иметь свой вес.

При помощи кнопки «Вопросы», доступной из формы «Электронные ресурсы», открывается форма «Вопросы для тестирования» со списком всех контрольных вопросов, имеющихся в информационной базе. На закладке «Рубрикаторы» этой формы содержится список рубрикаторов.

С помощью кнопки «Создать» открывается форма создания нового рубрикатора, в которой необходимо указать название создаваемого рубрикатора. Темы рубрикатора создаются кнопкой «Создать» в поле «Элементы рубрикатора».

Кнопками «Добавить» или «Подбор», расположенными в поле «Элементы рубрик», в тему рубрикатора добавляются вопросы из справочника «Вопросы для тестирования». Кнопкой «Добавить» можно добавить один вопрос, а кнопка «Подбор» позволяет добавить несколько вопросов последовательным кликом по их названиям. Один вопрос может быть добавлен в несколько тем рубрикаторов. После того, как вопрос добавлен в тему, можно установить вес этого вопроса в данной теме рубрикатора. Для этого необходимо кликнуть по названию вопроса в поле «Элементы рубрик», а потом в открывшейся форме вопроса указать его вес.

Теперь, после заполнения вопросами тем рубрикаторов, можно создать тест подбором вопросов из тем рубрикаторов. Например, «Итоговое тестирование» из 12 вопросов, которые выбираются по два вопроса из шести тем рубрикаторов.

Теория

Создать новый элемент теории можно с помощью кнопки «Создать->Теория» на форме электронного курса. При нажатии на эту кнопку открывается форма создания элемента теории. У нового элемента обязательно к заполнению поле «Наименование».

На закладке «Элементы» формы теории представлен состав теории из отдельных страниц, файлов и вопросов, а также папок с ними.

На закладке «Настройки» указывается тип навигации по страницам теории. Возможны следующие варианты:

  • «Последовательно» – переход к странице осуществляется нажатием на кнопку с ее номером.
  • «Выпадающее меню» – из наименований страниц программа формирует выпадающее меню. Меню раскрывается наведением на него мыши. Для перехода к странице нужно выбрать пункт меню с ее наименованием.
  • «Переключатели» – способ навигации аналогичен способу  навигации «Последовательно». Отличие в том, что страницы представлены не номерами, а своими наименованиями.
  • «Слайды» - навигация осуществляется последовательно путем плавного перехода с одной страницы на другую. Данный тип навигации используется, когда требуется, с одной стороны, разделить элементы теории на страницы, а с другой стороны, обеспечить максимально быстрое переключение между ними.
  • «Без навигации» – страницы размещаются последовательно друг под другом, без выделения разделов заголовками.
