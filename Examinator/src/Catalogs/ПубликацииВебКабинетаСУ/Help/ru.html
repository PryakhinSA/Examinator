Веб-кабинет позволяет организовать доступ к электронным ресурсам посредством веб-браузера с компьютера или мобильного устройства (iOS или Android планшета, фаблета, смартфона). Главные отличия веб-кабинета от тонкого клиента и веб-клиента платформы 1С:Предприятие 8 заключаются в его технической реализации. Веб-кабинет представляет собой веб-сайт (веб-приложение) с открытым исходным кодом, который можно использовать как самостоятельно, так и интегрировать в существующее веб-окружение. Функционально веб-кабинет предназначен для обучающихся и преподавателей и содержит необходимый им функционал, идентичный тому, что доступен для этих ролей пользователей в тонком и веб-клиенте.

Веб-кабинет размещается администратором на веб-сервере. Для его использования необходимо провести три этапа настройки. На первом этапе выполняется установка: выгрузка файлов на веб-сервер и формирование в информационной базе специальных данных в формате XML/JSON. На втором этапе выполняется публикация электронных ресурсов для использования в веб-кабинете. На третьем - создается экземпляр публикации веб-кабинета на веб-сервере с заданными настройками.

Публикацию веб-кабинета рекомендуется создавать при помощи мастера, который вызывается из формы «Публикации веб-кабинета» при помощи команды «Мастер публикации». Важно понимать, что мастер является только помощником для создания публикации веб-кабинета, но не выполняет ее самостоятельно. Он содержит инструкции по созданию публикации, а также результатом его работы является запись в справочнике «Публикации веб-кабинета», которая содержит все настройки созданной публикации. В будущем их можно будет отредактировать без прохождения всех этапов создания публикации.

Опытные пользователи могут создавать публикации без помощи мастера, используя команду «Создать» в форме «Публикации веб-кабинета», и выполнять действия на веб-сервере самостоятельно.

Для понимания работы мастера рассмотрим структуру публикации, ее составные части. Публикация веб-кабинета заключается в создании следующих каталогов и файлов (иерархия списка соответствует структуре вложенности каталогов и файлов и порядку их создания):

  1. Каталог публикации. Произвольный каталог на веб-сервере, в котором содержатся все составные части публикации. Допускается использовать корень веб-сервера в качестве каталога публикации.
    1. Каталог с опубликованной на веб-сервере информационной базой. Для работы веб-кабинета достаточно опубликовать веб-сервис «Электронные ресурсы» и http-сервисы «Обучение» (кроме конфигурации Конструктор курсов в виду его отсутствия в этой конфигурации), «Пользователи» и «Электронные ресурсы». Для доступа к этим сервисам веб-кабинета необходимо в информационной базе создать пользователя и назначить ему роль «Веб-кабинет - служебный пользователь». Мастер позволяет это сделать автоматически. Тонкий клиент и веб-клиент базы можно не публиковать. Можно создавать несколько публикаций одной базы, например, одну для доступа пользователей, а другую - исключительно для работы веб-кабинета. При необходимости допускается создавать данный каталог вне каталога публикации, но обязательно на данном веб-сервере.
    2. Виртуальная ссылка на каталог с файлами веб-кабинета, который был создан на первом шаге в процессе копирования файлов веб-кабинета. Виртуальная ссылка (alias) позволяет задать фактический адрес публикации, при этом физически файлы будут размещаться в общем каталоге.
    3. Виртуальная ссылка на каталог с публикациями электронных ресурсов (v8res), который автоматически создается в общем каталоге на втором этапе. Виртуальный каталог создается аналогично каталогу с файлами веб-кабинета., С его помощью пользователи получают доступ к электронным ресурсам, опубликованным в общем каталоге.
    4. Виртуальная ссылка на каталог основного тома файлов (кроме конфигурации Конструктор курсов в виду отсутствия данной функциональности). Данный каталог является виртуальной ссылкой на каталог основного тома хранения файлов. Используется только в том случае, если файлы хранятся в томах. При использовании веб-кабинета это является предпочтительным способом хранения файлов. Если том не настроен, то файлы будут выгружаться из информационной базы в каталог с файлами веб-кабинета, тем самым будет происходить увеличение нагрузки на веб-сервер и увеличение необходимого места на диске. При использовании небольшого количества файлов том хранения можно не использовать или же создать его позднее, когда файлов станет больше.
    5. Файл настроек (v8-settings.js), который содержит все настройки, необходимые для работы веб-кабинета. Формируется на последнем этапе работы мастера установки. Данный файл может быть в любой момент сформирован при помощи команды «Получить файл настроек» из формы «Публикации веб-кабинета».

Процедура публикации веб-кабинета заключается в создании на веб-сервере указанных выше каталогов и файлов в том порядке, как они указаны в списке.

Обратите внимание, что для открытия веб-кабинета необходимо открыть виртуальный каталог с файлами веб-кабинета, но не каталог публикации. Например, если каталог публикации называется «edu» и размещен в корне веб-сервера, то в нем будет каталог с файлами веб-кабинета, например, «cabinet», тогда для открытия веб-кабинета следует перейти в каталог http://my_server/edu/cabinet/.

Если необходимо сократить адрес веб-кабинета до минимального, то необходимо настроить редирект при открытии каталога публикации на каталог с файлами веб-кабинета. В каталоге публикации можно разметить другие собственные сайты или файлы, тем самым встроив веб-кабинет в общую веб-среду организации. Если в качестве каталога публикации выбран корень веб-сервера, то нельзя разместить более одной публикации на этом веб-сервере. Мастер содержит подробные рекомендации по созданию и настройке составляющих каталога публикации. После завершения работы мастера в справочнике «Публикации веб-кабинета» будет создана запись. На основе этой записи необходимо сформировать файл настроек и скопировать его в каталог публикации. На этом настройка веб-кабинета завершается.

Обратите внимание, что мастер предлагает создать или указать только самые необходимые настройки. В последующем можно открыть элемент справочника «Публикации веб-кабинета», изменить дополнительные настройки и обновить файл настроек (v8-settings.js) в каталоге публикации для их использования в веб-кабинете.

Приведем некоторые технологические замечания, которые могут быть полезны в случае, если потребуется более подробно изучить устройство веб-кабинета, например, с целью его интеграции или расширения.

Технологически веб-кабинет представляет собой клиентское (front-end) приложение на основе технологий HTML, JS, CSS, размещенное на веб-сервере. В качестве серверной (back-end) среды используется платформа 1С:Предприятие 8. Взаимодействие между клиентской и серверной частями выполняется посредством веб-сервисов и http-сервисов.

Клиентская часть НЕ требует наличия на веб-сервере дополнительных сред, вроде JAVA, PHP, ASP и др. Она является самодостаточным приложением и напрямую взаимодействует с 1С:Предприятие 8. В процессе взаимодействия для аутентификации на сервере 1С:Предприятие 8 используется один служебный пользователь, имеющий только одну служебную роль «Веб-кабинет - служебный пользователь». Важно! Логин и пароль данного пользователя будет указан в файле настроек и может быть получен любым пользователем веб-сервера. Следует внимательно следить, чтобы данный пользователь имел только данную роль и никакую другую. В этом случае публикация будет безопасна, так как не позволит злоумышленнику выполнить какие-либо действия в обход системы аутентификации веб-кабинета.

Права фактических пользователей веб-кабинета и процедура их аутентификации реализована на уровне прикладного кода сервисов 1С:Предприятие 8 и клиентской части. Проверка корректности выполняется для каждого запроса на уровне прикладного кода конфигурации.

На уровне платформы реализованы сервисы, которые позволяют получать или записывать данные при взаимодействии с клиентской частью. Обратите внимание, что данные, которые передаются клиентской части из базы данных, берутся не из прикладных объектов конфигурации, а предварительно формируются в формате XML или JSON, сохраняются в специальных таблицах и при запросе от клиента отдаются ему сразу, будучи уже заранее сформированными. Такой подход позволяет сделать работу пользователя быстрой, но накладывает ограничение на скорость обновления данных, доступных веб-кабинету. Также данный подход позволяет изолировать функциональность сервисов веб-кабинета от прикладных объектов.

В процессе работы в конфигурации формируются задания на обновление данных веб-кабинета. Задания выполняются при помощи регламентного задания «Обновление данных веб-кабинета». Для успешной работы веб-кабинета данное регламентное задание должно быть включено. Список заданий, которые сформированы, но еще не выполнены регламентным заданием можно посмотреть при помощи команды «Задания на обновления данных» в форме «Администрирование веб-кабинета». Помните, что данные веб-кабинета всегда будут обновляться с некоторой задержкой, которую можно регулировать при помощи расписания выполнения регламентного задания «Обновление данных веб-кабинета». Рекомендуемый интервал работы задания – 30-90 сек. Слишком малый интервал будет увеличивать вероятность ненужных блокировок данных, а слишком большой приведет к ухудшению комфорта работы организаторов обучения.

Обратите внимание, что для работы системы обновления учебных данных (кроме конфигурации «Конструктор курсов») требуется обязательно включить регламентное задание «Перепроведение документов учебного процесса». Оно необходимо для фонового обновления подчиненных документов. Например, при изменении параметров обучения данные каждого обучающегося будут обновлены этим регламентным заданием.

Рассмотренные особенности веб-кабинета дают возможность не только использовать готовый продукт, но и при необходимости развивать его самостоятельно, интегрируя в существующие информационные среды.

Клиентская часть имеет многостраничную архитектуру, которая позволяет добавлять собственные страницы. Функциональность взаимодействия с серверной частью вынесена в отдельный модуль, что позволяет написать свой API для своей архитектуры. Устройство серверной части позволяет вынести данные, необходимые для веб-кабинета, за пределы 1С:Предприятие, например, на сайт, тем самым еще больше сократив нагрузку на сервер. Встроенная система аутентификации тоже не является единственным возможным вариантом. На уровне веб-кабинета реализована поддержка CAS-сервера единой аутентификации. Также возможна реализация собственной логики аутентификации.

В целом, веб-кабинет позволяет создать готовый клиентский интерфейс, обладающий всеми характеристиками современного веб-ресурса, предназначенного для организации электронного обучения.
