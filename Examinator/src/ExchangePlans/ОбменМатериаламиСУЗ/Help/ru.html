Конфигурация позволяет выполнять импорт и экспорт пакетов данных, содержащих один или несколько электронных ресурсов (либо фрагментов электронных ресурсов). Инструменты, позволяющие выполнять экспорт и импорт электронных ресурсов, доступны по команде «Обмен» на форме списка электронных ресурсов.

Для выполнения импорта пакета данных требуется перейти на закладку «Импорт (загрузка)» и использовать команду «Импорт». Далее необходимо выбрать файл с данными. Это может быть файл формата XML, полученный путем экспорта данных из конфигурации «1С:Электронное обучение» той же версии (см. раздел «Экспорт данных»). Например, в конфигурацию релиза 3.0.34.26 возможен импорт XML файла, полученного в результате экспорта из конфигурации «1С:Электронное обучение» релиза 3.0.34.31, но НЕ возможен - из конфигурации релиза 2.0.13.3 или 3.0.33.1.

Также поддерживается импорт пакетов данных формата SCORM 2004 (полное название формата: SCORM 2004 4th Edition). Подробная информация о формате SCORM 2004 доступна по адресу: http://www.adlnet.gov/scorm/scorm-2004-4th. Рассмотрим подробнее настройку импорта пакетов SCROM.

При импорте пакета в формате SCORM 2004 после открытия файла будет доступна форма с выбором дополнительных параметров импорта:

  • «Импортировать как один ресурс». Установка данного флажка позволяет гарантировать работоспособность пакета SCORM после импорта вне зависимости от того, прописаны ли все зависимости отдельных ресурсов (resources) пакета. Если точно известно, что пакет SCORM включает множество независимых ресурсов (resources) и возможно их использование по отдельности, то флажок можно снять. В противном случае рекомендуется устанавливать флажок.
  • «Запомнить импортированные элементы». Установка флажка позволяет избежать дублирования ресурсов при повторном импорте пакета, например, новой версии. Если требуется намеренно создать копию импортируемого пакета, то флажок можно снять.
  • «Импортировать только ресурсы». Позволяет импортировать только ресурсы пакета (resources) без создания общего электронного ресурса (organizations). Установка флажка может быть полезной, если предполагается создание собственных электронных ресурсов из поставляемых в пакете отдельных объектов.
  • «Импортировать все ресурсы как активные». При установке флажка все импортируемые ресурсы будут отмечены как SCO (подробнее см. описание стандарта SCORM). Также этот флажок следует установить, если разработчик ошибочно не указал тип ресурсов в своем пакете.
  • «Создавать новую версию». В некоторых случаях, если были сделаны изменения в ранее импортированных данных, можно при повторном импорте пакета создать их новую версию. Установка флажка позволит не потерять сделанные изменения.
  • «Поставщик». Позволяет задать поставщика пакета данных.

 

Поддерживается экспорт пакетов данных в формат XML для последующей загрузки в другую информационную базу, созданную в той же версии программного продукта линейки «1С:Электронное обучение». Для выполнения экспорта электронных ресурсов требуется перейти на закладку «Экспорт (выгрузка)» и при помощи кнопки «Создать» сформировать новый пакет данных.

В новом пакете данных требуется ввести его название, а потом на закладке «Элементы для экспорта» указать один или несколько элементов для экспорта. Допускается выбор электронных ресурсов и/или фрагментов электронных ресурсов, названных «Элементы содержания». При выборе фрагмента выгружаемый пакет будет содержать только указанный фрагмент и подчиненные ему элементы. Допускается выбор групп элементов содержания электронного ресурса. В этом случае будут экспортированы все элементы, входящие в группы, в том числе и подчиненные группы.

Механизм обмена данными можно использовать для формирования дистрибутивов электронных курсов, которые будут автоматически устанавливаться на компьютере пользователя. Данная технология используется для создания тиражируемых электронных курсов, в том числе совместно выпускаемых разработчиками и фирмой «1С». С предложениями в области создания и распространения коммерческих электронных курсов обращайтесь в фирму «1С» по адресу e2b@1c.ru.
