Группа настроек «Просмотр объектов» содержит настройки, которые влияют на отображение объектов электронного ресурса.

Флажок «Использовать дополнительные материалы пользователей» позволяет включить возможность добавления пользователями к объектам электронных ресурсов собственных материалов. После установки этого флажка в форме списка электронных ресурсов появятся команды для управления дополнительными материалами Установка флажка «Показывать общие дополнительные материалы после проверки» позволяет проверять те материалы, которые пользователи прикрепляют к электронным ресурсам, перед тем, как они станут доступны другим пользователям.

В том случае, если пользователь изменяет тиражные электронные курсы и они обновляются поставщиком, следует настроить правила отображения объектов, имеющих несколько версий. Для этого устанавливается флажок «Показывать последнюю версию». В этом случае будет показываться последняя по номеру версия объекта от предпочитаемого поставщика. Поставщик данных указывается в поле «Предпочитать поставщика». В случае если у объекта нет ни одной версии от данного поставщика, будет показана последняя сохраненная версия. Если флажок не установлен, то при отображении учебного объекта будет показываться последняя сохраненная версия независимо от ее номера и поставщика.

Обратите внимание, что при указании предпочитаемого поставщика данный поставщик будет по умолчанию подставляться для вновь создаваемых объектов электронных курсов. Указание поставщика по умолчанию является обязательным условием для авторов тиражных курсов.

Группа настроек «Разработка» содержит настройки, которые влияют на инструменты разработчика электронных ресурсов.

Флажок «Изменять данные только предпочитаемого поставщика» позволяет предотвратить редактирование данных других поставщиков. Это может быть полезно при адаптации приобретенных курсов, которые периодически обновляются. При обновлении электронного курса другого поставщика все данные этого курса будут обновлены, а сделанные в них изменения утеряны. Данные других поставщиков при обновлении не удаляются. Подробнее см. главу 4 «Дополнительные возможности конструктора электронных курсов».

Поле «Максимальный размер файла» ограничивает размер файла (в байтах), который разработчик электронных курсов может загрузить в информационную базу. Если поле не заполнено, то разрешена загрузка файлов любого размера. В поле «Разрешенные расширения файлов» указываются те расширения файлов, которые разработчик может использовать в электронных ресурсах. Если поле не заполнено, то разрешены любые файлы.

Группа настроек «Общий каталог» позволяет настроить общий каталог на веб-сервере, который необходим для корректной работы веб-клиента платформы 1С:Предприятие 8 и веб-кабинета. Необходимость настройки общего каталога на веб-сервере обусловлена тем, что электронный ресурс может содержать файлы произвольных форматов (html, swf, java, png, avi, doc и т.п.), доступ к которым из браузера не возможен напрямую из базы 1С:Предприятие 8. При использовании тонкого клиента необходимые файлы автоматически копируются (кэшируются) на компьютер пользователя во временный каталог (кроме случая, когда установлен флажок «Использовать общий каталог в тонком клиенте»).

При использовании веб-клиента или веб-кабинета требуется выполнить дополнительную настройку веб-сервера и описываемой конфигурации, чтобы обеспечить возможность доступа пользователей к файлам электронных ресурсов. Настройка выполняется в два этапа:

  1. Создание на веб-сервере произвольного общего каталога;
  2. Указание в настройках конфигурации пути к этому каталогу в поле «Путь к общему каталогу» и его веб-адреса (URL) в поле «Адрес общего каталога».

На веб-сервере должен быть создан произвольный каталог, например, «cache». При его создании необходимо учесть следующие особенности:

Веб-адрес хоста каталога должен иметь тот же адрес хоста, что и адрес хоста информационной базы. Например, если база доступна по адресу «http://1c.ru/demoBase», то веб-адрес каталога может быть следующий: «http://1c.ru/cache». Допускается указание адреса без хоста, например «/cache».

  • Имя каталога не должно начинаться с тех же символов, что и адрес каталога информационной базы. Для рассмотренного выше примера не рекомендуется создавать каталог «http://1c.ru/demoBaseCache».
  • Каталог не должен быть размещен в каталоге, где опубликована конфигурация, например, для указанного выше примера нельзя создавать каталог «http://1c.ru/demoBase/cache».
  • Сервер должен быть настроен так, чтобы файлы из этого каталога могли отображаться пользователям в браузере без дополнительной авторизации (обычно это является стандартной настройкой веб-сервера, и дополнительных действий выполнять не требуется).
  • Следует учесть, что каталог кэша может содержать файлы,   которые   не   должны   быть   доступны  некоторым  категориям пользователей,   поэтому   рекомендуется   отключать   возможность  просмотра его содержимого из браузера.
  • Сервер «1С:Предприятие» должен иметь права на запись в каталог кэша.
  • Если рабочие процессы сервера размещены на различных  компьютерах,  то  все  они  должны  иметь  доступ  к данному каталогу. Допускается указание сетевого пути к каталогу.

После того, как общий каталог создан, необходимо указать конфигурации путь к этому каталогу и его веб-адрес. Для этого необходимо открыть форму настройки электронных ресурсов и  заполнить поля: «Путь к общему каталогу» и «Адрес общего каталога».

Допускается указание адреса каталога без указания хоста, например, для рассмотренного примера можно указать просто «/cache». После записи настроек требуется перезапустить программу.

Если после выполнения действий по настройке общего каталога появляются ошибки при просмотре электронных ресурсов, то для диагностики причин необходимо выполнить следующие шаги:

  • Зайти в общий каталог и проверить, имеются ли в нем файлы. Если каталог пуст, то причиной ошибки может быть:
    • В форме настроек информационной базы неверно указанный путь к общему каталогу.
    • Сервер «1С:Предприятие» не имеет прав на запись файлов в общий каталог.
  • Отобразить в браузере произвольный файл из общего каталога. Например, можно в браузере (без запуска 1С) указать путь к файлу изображения: «http:\\<адрес хоста>\<имя каталога>\v8cache_<номер версии>\res_common\img\ok32.png». В браузере должна отобразиться картинка. Если картинка не отобразилась, то возможны следующие причины ошибки:
    • Файла не существует. Необходимо вернуться к п.1 и проверить существование файла.
    • Сервер требует дополнительной авторизации для просмотра файла. Необходимо настроить веб-сервер для публичного доступа к файлам общего каталога.

Допускается удаление каталогов «v8cache_N», где N – номер версии каталога, из общего каталога для оптимизации используемого дискового пространства или в случае возникновения ошибок. Можно оставлять каталог только с последним номером версии.
