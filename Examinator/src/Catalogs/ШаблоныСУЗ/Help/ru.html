Шаблоны оформления основаны на технологии XSLT версии 1.0. Также применяются следующие технологии и стандарты: HTML, CSS, JavaScript.

Шаблон может включать дополнительные подчиненные шаблоны, которые отвечают за отображения подчиненных элементов объекта.

Внимание! Изменять шаблоны по умолчанию нельзя. При очередном обновлении конфигурации все изменения будут утеряны.

Для того чтобы создать собственный шаблон, можно использовать один из следующих вариантов:

  • Создать полный новый шаблон объекта и указать его для тех объектов, к которым он должен быть применен.
  • Добавить собственный подчиненный шаблон, который будет заменять один из существующих. Новый шаблон заменит существующий в том случае, если его поставщик указан как предпочитаемый в настройках информационной базы.
  • Создать новый шаблон объекта, который будет дополнять существующий по умолчанию.

Рассмотрим вариант 3 подробнее. Предположим, что нам требуется на страницах увеличить шрифт. Для этого можно сделать шаблон, который будет дополнять шаблон по умолчанию, и применить его к нужным страницам.

Создадим шаблон страницы, указав в реквизите «Вид» значение Страница, так как мы создаем шаблон для страниц. В поле «Дополняет шаблон» укажем шаблон по умолчанию для страниц. В качестве шаблона представления укажем:
 
<xsl:template name="template_36d6eebf-39d8-11dc-aac4-005056c00008"></xsl:template>

Идентификатор шаблона требуется скопировать из поля Идентификатор на закладке Дополнительно, добавив перед ним «template_». На закладке «Скрипт» указываем аналогичный текст. В более сложных шаблонах можно переопределить шаблоны (template) элементов объекта и вызвать их применение из своего шаблона. Для реализации этих возможностей потребуется знание технологии XSLT.

В нашем случае на закладке «Стили» укажем:

p {font-size: 24px;}

Откроем страницу, шрифт которой надо изменить, и заполним поле Шаблон, указав в нем наш новый шаблон. Таким образом можно создавать новое оформление для электронных ресурсов, вставлять необходимые скрипты, изменять терминологию и сообщения.
